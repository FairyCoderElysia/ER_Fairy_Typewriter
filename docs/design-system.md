# ER Fairy Typewriter 设计系统

这份文档用于约束后续页面和调试面板的视觉一致性。项目目标不是做花哨门户，而是做一套轻量、清晰、适合学习搜索引擎的界面。

## 设计目标

- 让搜索、调试、评测、抓取状态、候选源和调度器界面看起来像同一个项目。
- 让信息密度高，但不拥挤。
- 让用户一眼分清结果、来源、筛选、解释和错误状态。
- 保持实现简单，优先用原生 CSS 和现有模板完成。

## 品牌基调

- 关键词：轻量、二次元、克制、可学习、可解释。
- 视觉感受：不是资讯站，不是营销页，而是一个可以反复试验搜索效果的实验室。
- 品牌色：`#d9488f`，只用于标题焦点、主按钮、当前选中状态和关键标记。

## 字体

- 中文正文：优先系统中文字体，或 `Noto Sans SC`。
- 英文和代码：系统无衬线字体或等宽字体。
- 调试内容：分数、token、JSON、代码片段使用等宽字体。

建议字体栈：

```css
font-family: "Noto Sans SC", "PingFang SC", "Microsoft YaHei", sans-serif;
```

| 来源 | `#0f766e` | 来源、站点、来源标签 |
| 成功 | `#16a34a` | 成功状态和可用状态 |

## 间距

统一使用 4 的倍数：

```text
4 / 8 / 12 / 16 / 24 / 32
```

规则：

- 页面大分区之间至少留 `24px`。
- 卡片内部常用 `12px` 或 `16px`。
- 标题和说明之间常用 `8px`。
- 结果条目之间常用 `16px`。

## 组件

### 搜索框

- 必须有明确 label 或无障碍名称。
- 输入框和按钮应放在同一行或同一组。
- 默认不做花哨装饰，优先可用性。

### 按钮

- 主按钮：品牌色背景。
- 次按钮：浅色背景、边框。
- 危险按钮：仅用于明确破坏性动作。
- 按钮文案尽量写动作，如“搜索”“解释排序”“重建索引”“运行评测”。

### 卡片

- 只用于重复内容、分数解释、状态面板。
- 不要把整个页面做成一层层大卡片。
- 卡片圆角保持小，优先 `8px` 或更小。

### 标签

- 用于分类、来源、状态、字段命中。
- 同一页面标签样式要一致。

### 状态面板

- Loading：提示正在加载。
- Empty：提示空结果，并给出下一步建议。
- Error：说明失败原因和重试方式。
- Partial：保留已成功内容，同时显式标记缺失部分。

## 可访问性

- 所有输入、按钮、链接都要有清晰的 focus 状态。
- 文字和背景对比度至少满足 WCAG AA。
- 移动端点击目标不小于 44px。
- 结果页支持键盘 Tab 顺序浏览。
- 不要只靠颜色表达状态。

## 页面规则

- 首页：只保留一个搜索框和清晰品牌名。
- 结果页：搜索框、结果列表、筛选、解释入口要一眼可见。
- 调试页：信息密度可以更高，但要有清楚层次；`/debug` 总览页应把搜索解释、索引状态、Redis、抓取运行、候选源和调度器入口放在一起。
- 索引状态页：优先显示文档数、token 数、最近重建时间、失败记录和当前索引后端，例如 `memory`、`redis-zset-like`、`redis` 或 `meilisearch`。
- 抓取状态页：优先显示最近运行、保存数量、错误数量和错误明细，部分失败要保持可读。
- 候选源页：已配置源、候选源、`source_type`、关键默认配置和操作入口要能快速扫读。
- 调度器页：优先显示是否启用、全局间隔、每源上次运行、下次运行、源级间隔和最近结果。

## 实现建议

- 优先复用现有 `static/style.css`。
- 新页面先对齐这份文档，再写布局。
- 若与现有样式冲突，先改文档再改代码。
